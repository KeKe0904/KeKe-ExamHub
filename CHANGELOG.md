# 更新日志

## [1.1.0] - 2026-06-26

### 新增
- **环境管理页面** — 一键检测/更新运行环境（Node.js、npm、PM2、Nginx、MariaDB、Git），实时日志反馈
- **服务器监控页面** — CPU / 内存 / 磁盘实时使用率、系统负载、进程信息
- **Cookie 工具** — JWT Token 持久化存储，刷新页面无需重新登录
- **install.sh 版本自动检查** — `git fetch` 对比 GitHub，发现新版本自动 pull + 显示变更日志

### 优化
- **install.sh 大重构** — 动态路径（脚本所在即项目目录）、非 root 模式、apt lock 冲突自动等待、部署后健康检查
- **Node.js 升级到 24 LTS** — 旧版（18/20/22）自动覆盖安装到最新 24.x
- **npm 自动升级** — 每次部署自动 `npm install -g npm@latest`，确保 npm 为最新版
- **环境页面轮询优化** — 组件卸载时自动清理所有定时器，不再卡住
- **一键更新逻辑优化** — 仅更新有可用更新的组件（而非每次都全部更新），动态统计完成步骤
- **部署脚本环境检测** — 跳过已是最新版本时不重复构建，构建产物缺失时自动重建

### 修复
- 修复 15 处 TypeScript `TS18048` 构建错误（`task` 可能为 `undefined`）
- 移除 environment.ts 中硬编码的 `/opt/examhub` 路径，改用动态 `PROJECT_ROOT`
- 修复前端环境页面轮询任务结束后 `setInterval` / `setTimeout` 未清理的问题

## [1.0.5] - 2026-06-26

### 修复
- 修复生产环境前端 API 请求路径错误（`http://localhost:3000/api` → `/api`）
- 修复部署后管理员登录失败的问题（请求未通过 Nginx 反向代理）

### 优化
- 前端 API 基础路径改为相对路径 `/api`，适配 Nginx 反向代理
- Vite 开发服务器添加 `/api` 代理配置，开发环境统一使用相对路径
- 部署脚本 `install.sh` 构建前端时显式设置 `VITE_API_BASE_URL=/api`
