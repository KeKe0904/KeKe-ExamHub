# 学习工作 · KeKe ExamHub — 学校考试信息数字化管理平台

> 把教室里那块上万元的大屏，从"下课就黑屏"变成"考试信息神器"。

---

## 一、这是个什么东西？

**KeKe ExamHub** 是一个学校考试信息管理平台。说人话就是：**把考试信息搬上网，老师、学生、教室大屏都能看，再不用满楼贴通知纸了。**

打开浏览器就能用，不用买新硬件，不用装 APP。教室里现有的那块大屏一体机，输个网址就能变成考试信息屏。

### 五种身份，各看各的

| 谁 | 在哪用 | 能干啥 |
|----|-------|-------|
| **路人** | 首页（不用登录） | 看考试列表、看学校公告 |
| **管理员** | 后台 | 录考试、管教室、发公告、看日志——啥都能干 |
| **老师** | 教师端 | 看自己要监考哪场、班里有谁 |
| **学生** | 学生端 | 看自己啥时候考、在哪个教室 |
| **教室大屏** | 教室端 | 全屏显示考试倒计时，字大得最后一排都看得清 |

## 二、为啥要做这个？

### 起因：教室门口那张通知纸

> "每次考试最烦的是啥？不是复习，是不知道具体几点考、在哪个教室、谁监考。那张通知纸贴门口，风一吹就没了。老师口头说一句，没听清就完了。"
>
> —— 我自己就是学生

然后我看了一眼教室里那块**86 吋的大屏一体机**。这玩意儿上万块买的，希沃、鸿合这些牌子，集采价 8700 到 22500 元一台。结果呢？上课投屏，下课黑屏。**考试信息这么重要的事，它居然啥都不显示。**

要是能让它自动显示考试信息，字放大，加个倒计时，不比贴张纸强？

### 我发现的 5 个真问题

**1. 通知纸太容易丢**

几十间教室，一次考试，教务老师得跑遍整栋楼手写通知。时间改了？重写。字潦草？学生看不清。

**2. 上万的大屏当摆设**

希沃、鸿合一体机，上课用，下课黑屏。考试这么重要的场景，没软件管。

**3. 信息传话游戏**

教务处 → 教师群 → 班主任 → 班级群 → 家长 → 学生。传一层丢一点。最后学生在群里问八百遍"几点考来着"。

**4. Excel 排表靠人眼防冲突**

同一时间同一教室排重了？同一个老师分到两个考场？Excel 不会提醒你，全靠盯。

**5. 学生总是最后知道的**

考试是给学生考的，但学生获取信息的渠道最不靠谱——纸会丢，话会忘，黑板会擦。

### 我查了市面上有的产品

开发前我跟 TRAE 一起调研了一圈，发现：

| 产品 | 问题 |
|------|------|
| 希沃云班牌 | 要钱，要买他家硬件 |
| 鸿合电子班牌 | 要钱，要买他家硬件 |
| AOLSEE 傲视电子班牌 | 要钱，要买他家终端 |
| WHBestSoft 智慧班牌 | 要钱，功能还臃肿 |
| 龍威 5G 云班牌 | 要钱，要装专用硬件 |

> **结论**：市面上全是要钱、要专用硬件的。**没有一款免费、开源、专门用教室现有大屏显示考试信息的产品。** 这就是我要做的。

### 谁用？什么时候用？

| 谁 | 用啥看 | 多久看一次 |
|----|-------|-----------|
| 教务管理员 | 电脑 | 考前几天录数据 |
| 监考老师 | 手机 / 大屏 | 考试当天多次看 |
| 学生 | 手机 | 考前 24 小时频繁看 |
| 教室大屏 | 86 寸一体机 | 考试全程开着 |

## 四、用 TRAE 怎么开发出来的

> **从头到尾，一行代码都是在 TRAE IDE 里写的。** 下面讲讲几个关键节点。

### 4.1 开工：连"做啥"都是 TRAE 帮我想清楚的

我一开始只有个模糊想法："教室大屏应该能显示考试信息"。

先跟 TRAE 一起把市面上的竞品列出来，做成表格对比。**确认这个方向没人做过之后**，才开始搭架子。

跟 AI 说："我要做个 React 前端 + Fastify 后端 + MySQL 数据库的项目"。TRAE 直接帮我把项目骨架生成好了——目录、配置、入口文件，全有。

> 以为搭环境得两三天，**结果半小时就开始写正经功能了**。

### 4.2 数据库：从 8 张表长到 20 张表

报名那会儿我只想到 8 张表：管理员、考试、公告、设置、教学楼、注册码、教室、考试-教室关联。

后来每加一个功能，就跟 TRAE 商量要不要加表：

| 新功能 | 加了哪些表 |
|-------|-----------|
| 老师端 | teachers、teacher_roles、exam_invigilators |
| 学生端 | classes、students、exam_students |
| 安全 | ip_blacklist、classroom_login_logs、classroom_trusted_ips、admin_logs |
| 域名证书 | domains |
| 教室倒计时 | classroom_countdowns |

> 从 8 到 20，**每张表都是跟 AI 聊出来的**。

### 4.3 安全：五种身份互不串门

我最在乎的就是：**学生绝对不能进管理员后台，老师绝对不能看学生密码**。

跟 TRAE 设计了四套"门卫"：

```
管理员门卫  → 只让管理员进
教室门卫    → 只让教室端进
学生门卫    → 只让学生进
老师门卫    → 只让老师进
```

还做了几个安全细节：
- 账号不存在也跑一次密码校验，**防止有人用响应时间猜账号**
- 登录失败 5 次就锁一分钟
- 密钥太弱就进"只能安装"模式，防止线上裸奔

后来让 TRAE 帮我做了 39 次越权测试——用学生身份调管理员接口、用老师身份调学生接口……

> **39 次全部被挡在门外（403）。那一下挺爽的。**

### 4.4 教室大屏：调了好多版的视觉效果

教室端是整个项目最"有画面感"的部分。监考模式得全屏、字得大、时间快到了得变红。

跟 TRAE 改了好多版：

- 字号从小调到大，**最后一排能看清**
- 剩 5 分钟字变红 + 闪烁
- 进度条加渐变
- 一键全屏
- 30 秒自动刷新，不用人管

> 在一体机上全屏跑起来，效果比我想的还好。

![教室端监考大屏](https://gh-proxy.com/https://raw.githubusercontent.com/KeKe0904/KeKe-ExamHub/main/assets/screenshots/classroom-invigilation.png)

### 4.5 AI 小助手：能听人话，还分安全等级

管理员能跟 AI 说人话操作后台，比如："帮我查下周有几场数学考试"、"建一场英语期末"。

关键是**安全**——AI 不能啥都干。我设计了三档：

| 级别 | 数量 | 啥意思 |
|------|------|-------|
| **安全** | 13 个 | 只是查数据，直接干，不问 |
| **危险** | 14 个 | 要改数据的，弹红框，用户点确认才干 |
| **禁止** | 不给工具 | 删考试、重装系统这种，AI 直接说"不行，你自己去后台弄" |

技术上踩了一堆坑：流式响应、超时、结果太长截断、上下文记不住……都是跟 TRAE 一起解的。

![AI 助手](https://gh-proxy.com/https://raw.githubusercontent.com/KeKe0904/KeKe-ExamHub/main/assets/screenshots/ai-assistant.png)

### 4.6 安装脚本：本来想写个简单的，结果写了 1279 行

部署脚本是个意外大工程。本来想"写个小脚本得了"，结果跟 TRAE 一起迭代了 4 个版本：

| 版本 | 加了啥 |
|------|-------|
| v1 | 装 Node + Nginx + PM2 |
| v2 | 加域名和 SSL 证书 |
| v3 | 加错误处理和日志 |
| **v4** | 加断点续传、DNS 校验、SSL 四选项、严格错误模式 |

> **最难忘的 bug**：DNS 校验一直过不去，跟 TRAE 查了快一小时，**最后发现是 GNU grep 不支持 `\d`，得写成 `[0-9]`**。这种坑自己踩不知要多久。

![安装向导](https://gh-proxy.com/https://raw.githubusercontent.com/KeKe0904/KeKe-ExamHub/main/assets/screenshots/setup-wizard.png)

### 4.7 发版前安全大检查：TRAE 帮我揪出一堆漏洞

v1.2.0 发之前，我让 TRAE 给整个项目做了次安全体检。查出一堆我自己根本想不到的问题：

| 问题 | 多严重 | 怎么修的 |
|------|-------|---------|
| 证书申请代码可能被注入命令 | 严重 | 换成安全的调用方式 |
| 安装脚本里留了管理员明文密码 | 严重 | 装完就删 |
| 域名参数没校验，可能被坏人利用 | 高 | 加格式校验 |
| IP 获取方式可被伪造 | 高 | 换成统一的安全方式 |
| 公告内容没净化，可能被注入恶意脚本 | 高 | 加二次净化 |
| 分页参数传大了就报错 | 高 | 换数据库调用方式 |
| 注册码用随机数太弱可被猜 | 中 | 换成加密随机 |

> 从发现问题到修代码到重新验证，**全在 TRAE 里搞定，一条龙**。

### 4.8 踩过的坑

都是跟 TRAE 一起解决的：

1. **ESM 的 `__dirname` 没了** — 后端用了新模块系统，老变量不能用，换 `import.meta.url` 推导
2. **数据库分页参数坑** — `LIMIT ?` 传 JS 数字会报错，换个函数就好
3. **DNS 正则的 `\d` 不工作** — GNU grep 不认 `\d`，得用 `[0-9]`
4. **pm2 没装会崩** — 加了错误捕获
5. **安装向导建表建不全** — 一开始只建 9 张，缺一半，改成直接加载 SQL 文件
6. **SQL 里写死了数据库名** — 用户改库名就报错，删掉就行
