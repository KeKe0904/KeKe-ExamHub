## 一、项目简介

**KeKe ExamHub** 是一个面向中小学和考试机构的**考试信息管理平台**。核心目标很明确：**把考试信息从纸质通知搬到网上，让管理员、老师、学生、教室大屏都能实时看到，不再依赖满楼贴通知纸。**

打开浏览器就能用，无需购买额外硬件，无需安装客户端。教室里现有的希沃、鸿合等一体机，输入网址就能变成考试信息展示屏。

### 五种身份，各司其职

| 角色 | 入口 | 能做什么 |
|------|------|---------|
| **访客** | 首页（无需登录） | 浏览考试列表、查看学校公告 |
| **管理员** | 后台 | 录入考试、管理教室、发布公告、查看操作日志 |
| **教师** | 教师端 | 查看监考任务、班级学生信息 |
| **学生** | 学生端 | 查看个人考试安排（时间、地点） |
| **教室大屏** | 教室端 | 全屏显示考试倒计时，字号足够大，最后一排可见 |

### 核心功能

- **考试全流程**：创建考试 → 分配教室 → 指派监考老师 → 学生按班级关联 → 教室大屏实时倒计时

## 二、创作思路

### 灵感来源：教室门口的通知纸

> "每次考试最麻烦的不是复习，而是经常不知道具体时间和监考老师。通知纸贴门口，风一吹就没了。老师口头说一句，没听清就错过了。"
>
> —— 这是作为学生的真实感受

教室里那块**86 吋的一体机**，希沃、鸿合等品牌集采价约 8700-22500 元。然而上课投屏，下课黑屏。考试信息如此重要的场景，却没有专门的软件来支撑。

如果能让它自动显示考试信息，字号放大，加上倒计时，会比纸质通知可靠得多。

### 五个真实痛点

**1. 通知纸易丢失**

几十间教室，一次考试，教务老师需要跑遍整栋楼手写通知。时间改动需要重写，字迹潦草则学生无法辨认。

**2. 一体机闲置**

希沃、鸿合一体机价格上万元，但仅在上课时用于投屏，下课后黑屏。考试这一高频刚需场景缺乏专用软件。

**3. 信息传递链路过长**

教务处 → 教师群 → 班主任 → 班级群 → 家长 → 学生。每经过一层传递，信息失真的风险就增加一分。最终学生在群里反复询问确认。

**4. Excel 排表难以发现冲突**

同一时间同一教室是否排重？同一位老师是否被分到两个考场？Excel 不会自动提醒，全靠人工核查。

**5. 学生是最后获得信息的人**

考试信息的最终消费者是学生，但学生获取信息的渠道最不可靠——通知纸会丢失，口头传达会遗忘，黑板会被擦除。

### 竞品分析

开发前与 TRAE 一起调研了市面上的类似产品：

| 产品 | 现状 |
|------|------|
| 希沃云班牌 | 商业付费，需配套希沃专用硬件 |
| 鸿合电子班牌 | 商业付费，需配套鸿合专用硬件 |
| AOLSEE 傲视电子班牌 | 商业系统，需购买专用终端 |
| WHBestSoft 智慧班牌 | 商业付费，功能臃肿 |
| 龍威 5G 云班牌 | 需部署专用硬件终端 |

> **结论**：市面上的班牌产品均为商业付费、闭源、需专用硬件。**没有一款免费开源、专注于复用教室现有大屏来展示考试信息的产品。** 这正是 ExamHub 要填补的空白。

### 目标用户与使用场景

| 角色 | 使用设备 | 使用频率 |
|------|---------|---------|
| 教务管理员 | 电脑 | 考前几天录入数据 |
| 监考老师 | 手机 / 教室一体机 | 考试当天多次查看 |
| 学生 | 手机 | 考前 24 小时频繁查看 |
| 教室大屏 | 86 寸一体机 | 考试期间持续运行 |

### 4.1 项目启动：从竞品分析到骨架搭建

开发前先与 TRAE 一起做了竞品分析，把市面上的智慧班牌系统逐一列出对比，**确认这个细分场景确实空白之后**，才开始搭建项目。

向 AI 描述技术栈需求："React 前端 + Fastify 后端 + MySQL 数据库"。TRAE 直接生成了完整的项目骨架——目录结构、配置文件、入口文件一应俱全。

> 原本预计搭环境需要两三天，**实际半小时后就开始写业务功能了**。

### 4.2 数据库：从 8 张表扩展到 20 张表

报名时只规划了 8 张表：管理员、考试、公告、设置、教学楼、注册码、教室、考试-教室关联。

开发过程中每新增一个功能，就与 TRAE 讨论表结构设计：

| 新增功能 | 对应新增表 |
|---------|-----------|
| 教师端 | teachers、teacher_roles、exam_invigilators |
| 学生端 | classes、students、exam_students |
| 安全体系 | ip_blacklist、classroom_login_logs、classroom_trusted_ips、admin_logs |
| 域名 SSL | domains |
| 教室个性化 | classroom_countdowns |

> 从 8 张到 20 张表，**每一张都是与 AI 对话中确定的设计**。

### 4.3 权限安全：39 项越权测试全部返回 403

安全是项目的核心关注点。**学生绝不能进入管理员后台，教师绝不能查看学生密码。**

与 TRAE 一起设计了四套独立的认证中间件：

```
authMiddleware          → 只放行 role === "admin"
classroomAuthMiddleware → 只放行 role === "classroom"
studentAuthMiddleware   → 只放行 role === "student"
teacherAuthMiddleware   → 只放行 role === "teacher"
```

安全细节：
- 账号不存在时同样执行一次密码校验，**防止通过响应时间差猜测账号是否存在**
- 登录接口加速率限制，管理员 5 次/分钟
- JWT 密钥强度不足时进入"仅安装模式"，防止生产环境裸奔

后来让 TRAE 执行了一次全面越权测试：用学生身份调用管理员接口、用教师身份调用学生接口……

> **39 项越权测试，全部返回 403。**

### 4.4 教室大屏：反复打磨的视觉效果

教室端是整个项目视觉表现最强的部分。监考模式需要全屏、字号大、剩余时间不足时变红提醒。

与 TRAE 来回调整了多个版本：

- 字号逐步放大，**确保最后一排可见**
- 剩余 5 分钟变红 + 脉冲动画
- 进度条加渐变效果
- Fullscreen API 一键全屏

### 4.5 AI 助手：27 个工具 + 三级安全

管理员可以用自然语言操作后台，例如"帮我查下周有几场数学考试"、"创建一场英语期末考试"。

**三级安全体系**是核心设计：

| 级别 | 数量 | 行为 |
|------|------|------|
| **safe** | 13 个 | 纯查询操作，直接执行 |
| **dangerous** | 14 个 | 写入/修改操作，前端弹出确认框，用户确认后执行 |
| **forbidden** | 不提供工具 | 删除考试、重装环境等高危操作，AI 直接拒绝并引导手动操作 |

技术上解决了多个难题：SSE 流式响应、工具执行超时、结果过长截断、上下文窗口滑动……都是与 TRAE 一起逐一解决的。

### 4.7 安全审计：TRAE 协助发现并修复漏洞

v1.2.0 发布前，让 TRAE 对整个项目做了一次安全审计，发现多个之前未注意到的问题：

| 问题 | 严重程度 | 修复方式 |
|------|---------|---------|
| 证书申请代码存在命令注入风险 | 严重 | 改用 execFileSync 传参数数组 |
| 安装脚本中残留管理员明文密码 | 严重 | 安装完成后删除 |
| 域名参数未校验，存在路径遍历风险 | 高 | 增加正则校验 |
| IP 获取方式可被伪造 | 高 | 统一使用 getClientIp |
| 公告内容未净化，存在 XSS 风险 | 高 | 增加 DOMPurify 二次净化 |
| 分页参数过大时接口报错 | 高 | pool.execute 改为 pool.query |
| 注册码随机数强度不足 | 中 | 改用 crypto.randomInt |

> 从发现问题、分析原因到编写修复代码、重新验证，**全程在 TRAE 中完成**。

### 4.8 踩过的坑

均是与 TRAE 一起解决的：

1. **ESM 模块下 `__dirname` 不可用** — 后端使用 `"type": "module"`，改用 `import.meta.url` 推导路径
2. **mysql2 预处理 LIMIT 参数报错** — `pool.execute()` 配合 `LIMIT ?` 传 JS 数字会出错，改用 `pool.query()`
3. **DNS 正则 `\d` 不生效** — GNU grep ERE 不支持 `\d`，需使用 `[0-9]`
4. **pm2 未安装时崩溃** — `spawn("pm2")` 触发 uncaught error，增加 `.on("error")` 捕获
5. **安装向导建表不全** — 最初只建了 9 张表，改为直接加载 init.sql
6. **init.sql 中 USE 语句导致权限错误** — 写死的数据库名在用户自定义库名时报错，删除即可
