## 1. 创意名称 + 创意介绍

**创意名称：** KeKe ExamHub — 学校考试信息数字化管理平台

### 想解决什么问题

当前中国绝大多数中小学的考试信息传递仍停留在"粉笔+黑板"时代：

- 教务处老师在考试周需要逐个教室手写考试安排表，字体不统一、排版靠经验、信息容易写错。一旦考试时间或科目调整，就必须重新擦、重新写，耗时且低效。一所 36 间教室的学校，一次期中考试光手写通知就要跑遍整栋楼，往返耗时超过 2 小时。
- 考试信息通过家长群、班级群、教师群层层转发，"传话游戏"让信息逐渐失真，学生和家长反复私信确认。
- 教室里的交互式一体机（希沃、鸿合等品牌，尺寸 65~86 英寸，单价过万）90% 时间只充当"高清投影仪"——上课投屏 PPT，下课挂着静态电子班牌，数字化价值远未释放。

### 为什么会想到做这个

我自己就是学生。每次考试，最麻烦的不是复习，而是经常不知道具体时间和监考老师——那张贴在门口的通知纸太容易弄丢了，有时候老师也不会写黑板，全靠口头说，有的教室连个钟都没有，电脑上的时间小得坐在后排根本看不清。

于是我就想：教室里那块大屏幕为什么不能直接显示考试信息？科目、时间、监考老师，把字放到最大，再加一个倒计时。这就是 KeKe ExamHub 的起点。

### 大概是什么产品

一个全栈 Web 系统（网站形态），前端适配 PC / 平板 / 手机 / 教室一体机四种尺寸，后端一键部署、图形化安装向导初始化，零技术门槛即可上线使用。系统由三层入口构成：公众前台（学生/家长浏览）、教室端大屏（一体机自动展示+监考模式）、后台管理（管理员发布考试、分配教室、审核注册）。


## 2. 目标用户及痛点

### 面向哪些用户

| 用户角色 | 核心需求 | 使用设备 | 使用频率 |
|---------|---------|---------|---------|
| 教务处 / 管理员 | 统一发布考试、掌握全局安排、调度教室资源、审核教室端注册 | 电脑（后台管理系统） | 考试周前 1-2 次/天 |
| 监考老师 / 班主任 | 快速查看自己所负责教室的考试信息（时间、科目、注意事项） | 手机 / 教室一体机 | 考试当天多时段查看 |
| 学生 | 随时随地查询自己的考试时间、科目、地点、监考老师和注意事项 | 手机（浏览器访问） | 考前 24 小时高频访问 |
| 教室一体机 | 作为固定展示终端，自动显示本教室被分配的考试信息 + 实时倒计时 | 86 吋触控大屏（希沃/鸿合等） | 考试期间连续运行 |

### 在什么场景下使用

**场景 1｜考前准备期（T-7 天至 T-1 天）**

管理员登录后台 → 创建教学楼 → 批量生成注册码（`XXXX-XXXX` 格式）→ 为每场考试录入科目、日期时间、时长、地点、监考老师、注意事项 → 为考试分配教学楼和教室。公众首页自动展示考试汇总列表，支持按"即将开始/进行中/已结束"筛选。

**场景 2｜教室端激活**

教室一体机打开浏览器 → 注册页输入注册码、选择教学楼、填写教室号（如 301）、设置密码 → 提交后等待管理员审核。管理员在后台看到待审核列表，点击"通过"后该教室端方可登录。

**场景 3｜考试当天（T-0）**

每间教室的一体机登录教室端 → 自动显示本教室今天被分配的所有考试科目及时间区段 → 页面每 30 秒自动刷新，无需任何人工操作。对于没有一体机的教室，学生或老师也可以用手机浏览器直接访问公众首页查看。

**场景 4｜考试进行中（监考模式）**

教室端一键切换"监考模式"→ 超大字号全屏倒计时（`text-7xl`）、显示科目名称、已过/总时长进度条 → 剩余时间低于 5 分钟时自动触发红色脉冲预警，提示监考老师准备收卷 → 支持调用浏览器 Fullscreen API 一键全屏。

**场景 5｜考试临时调整**

若因特殊情况需要临时修改考试时间或更换教室，管理员在后台编辑后，所有教室端的考试列表和公众首页实时同步生效，无需重新发布通知，无需派人跑楼。

**场景 6｜日常教务管理**

管理员可随时维护教学楼信息（增/删/改）、生成或撤销注册码、按状态筛选教室端申请（待审核/已通过/已驳回）、管理学校公告、查看服务器运行状态和资源使用率。

### 当前痛点（如果没有这个产品，用户会面临什么）

**痛点 1｜信息一致性灾难**

手写通知在数十间教室分散张贴，字迹因老师不同而千差万别。一旦考试时间或教室调整，需要派人跑遍全楼修改每一张通知，往返耗时 20 分钟以上。信息出错率极高，曾有学校因手写通知时间写错导致全年级学生错过考试。

**痛点 2｜一体机资源巨额浪费**

价值上万甚至数万元的教室一体机，上课之外要么黑屏要么播放无意义的静态画面。考试信息展示是一个高频刚需场景，却没有任何专用软件来承接，一体机的数字化价值被彻底搁置。

**痛点 3｜沟通链路脆弱**

考试安排依赖教师群→班级群→家长群的"传话游戏"。每经过一层转发，信息就有一次失真风险。最终学生和家长需要频繁私信确认，大量重复答疑消耗班主任和教务处精力。

**痛点 4｜无法全局统筹**

管理员用 Excel 排表，无法直观判断同一教学楼内是否存在教室冲突（两场考试分配到同一教室同一时段）、同一监考老师是否被同时安排到两个考场、同一班级是否在同一时段被安排多门考试。

**痛点 5｜学生信息获取困难**

学生作为考试信息的最终消费者，获取途径却最不可靠。一纸通知风吹即散，口头传达听过即忘，黑板板书擦完即无。没有人永久留存，没有人实时更新，没有人在意坐在后排的学生能不能看清楚。


## 3. 价值与意义

### 效率提升（量化视角）

| 操作 | 传统方式 | KeKe ExamHub | 提升倍数 |
|------|---------|--------------|---------|
| 发布全年级考试安排 | 2 名老师、2 小时（手写 36 间教室通知） | 1 人、5 分钟（后台批量录入） | 24x |
| 修改考试时间 | 跑楼重写通知、拍照重发群（约 30 分钟） | 后台编辑→全局实时刷新（约 10 秒） | 180x |
| 学生查询考试信息 | 翻群聊记录/问同学/去公告栏看（2-5 分钟） | 打开网页即看到全部安排（5 秒） | 24x~60x |
| 教室端信息更新 | 人工逐个教室切换画面或手写 | 自动刷新（30 秒间隔），零人工 | ∞ |
| 教室端激活 | 无（没有这个功能） | 注册码注册→管理员审核，全程在线 | 新能力 |

### 社会价值

**降低教育数字化转型门槛**

项目完全开源免费（GitHub 公开仓库，MIT 协议）。对于经费紧张的乡镇中小学、偏远地区学校、民办教育机构而言，无需购买任何商业考试管理系统（市场年费通常在 3000 元至 5 万元不等），即可拥有完整的数字化考试信息管理能力。

**安全可控的注册码体系**

教室端采用注册码注册 + 管理员审核的双重准入机制。管理员批量生成 `XXXX-XXXX` 格式注册码（去除易混淆字符 I/O/0/1），每码一次性使用。注册码不设硬外键约束，由应用层保证数据一致性。管理员对每一间接入系统的教室端拥有完全控制权：通过、驳回或删除。从架构源头杜绝了非法终端接入和考试信息泄露的风险。

**零技术门槛的部署体验**

项目自研网页安装向导：服务器端一条脚本完成环境部署（自动安装 Node.js + Nginx + MariaDB + PM2），浏览器端图形化向导 4 步完成数据库创建、8 张表结构初始化、管理员账号设置。一个信息技术老师 10 分钟即可完成整套系统上线，无需阅读任何命令行文档。

**推动一体机设备价值觉醒**

让全国数百万间教室里的交互式一体机从"高清投影仪"升级为"数字考试终端"，在不增加任何硬件投入的前提下将设备利用价值提升数倍。这一思路同样适用于电子班牌、校园信息发布屏等校园数字终端。


## 4. 核心功能架构

### 系统三层入口架构

```
                          KeKe ExamHub
                               │
              ┌────────────────┼──────────────────┐
        公众前台              教室端大屏               后台管理
       (任何人访问)      (一体机终端 + 监考)        (管理员认证)
              │                   │                    │
    ┌─────────┼─────────┐   ┌─────┼─────┐    ┌────────┼────────┐
    │                   │   │           │    │        │        │
  考试列表          学校公告  学生查看   监考模式   考试管理   教室调度
  (3列网格)         (实时同步)  (大屏卡片)  (全屏倒计时)  (CRUD)    (教学楼+注册码)
    │                                   │    │        │        │
  状态筛选              │             5分钟预警  教室分配   教室端审核
  实时倒计时             │             进度条    (多对多)   (通过/驳回)
  搜索过滤               │            全屏API
```

### 公众前台功能模块

- **Hero 横幅** — 展示考试统计数据（考试总数/即将开始/进行中/已结束），装饰数学函数曲线背景
- **公告栏** — 管理员发布的公告实时同步到首页，支持置顶、长文展开/收起、一键关闭
- **考试列表** — 3 列响应式网格布局，每张卡片含科目名称、日期时间、考试时长、地点、监考老师、注意事项、实时倒计时
- **搜索筛选** — 按科目/地点/老师关键词搜索，按考试状态标签筛选
- **学校信息** — 展示学校名称、Logo、介绍信息
- **深色模式** — 自动跟随系统主题偏好

### 教室端功能模块

- **注册流程** — 输入注册码 → 选择教学楼 → 填写教室号 → 设置密码 → 提交等待审核
- **登录鉴权** — JWT Token 独立签发，与管理员 Token 互斥验证。未审核通过时显示"管理员审核中"提示，已驳回显示驳回原因
- **学生查看模式** — 深色背景大屏展示本教室被分配的考试信息，卡片式布局含科目、时间、时长、监考老师、注意事项、实时倒计时。页面每 30 秒自动刷新，已在进行的考试卡片显示绿色脉冲标识
- **监考模式** — 顶部考试标签选择器切换不同场次 → 超大倒计时（文字尺寸 `text-7xl`）→ 已过/总时长进度条 → 剩余 5 分钟自动触发红色脉冲预警 → 调用 Fullscreen API 全屏 → 2×2 信息网格展示时间/时长/老师/地点 → 注意事项区域醒目标记
- **教室-考试关联** — 通过 `exam_classrooms` 多对多关联表实现，教室端仅显示被管理员分配到的考试，确保信息安全不扩散

### 后台管理功能模块

- **仪表盘** — 考试数据概览统计（总数、各状态分布、近期考试列表）
- **考试管理** — 发布/编辑/删除考试，可视化日期时间选择器，关联教室分配（按教学楼分组选择，支持全选/单选/整组切换）
- **教学楼管理** — 创建/编辑/删除教学楼（教室端注册时选择），删除前自动检查是否有教室关联
- **注册码管理** — 批量生成 1~50 个注册码（`XXXX-XXXX` 格式，去除 I/O/0/1 易混淆字符），按使用状态筛选，支持删除未使用的注册码
- **教室端账号管理** — 查看注册申请列表（可按状态筛选：待审核/已通过/已驳回），审核通过/驳回操作，删除已注册教室端（自动释放关联的注册码），实时待审核数量提示
- **公告管理** — 发布/编辑/删除公告，支持置顶和上下架
- **环境管理** — 一键检测运行环境组件版本（Node.js / npm / PM2 / Nginx / MariaDB / Git），一键自动更新至最新版本，实时日志反馈
- **服务器监控** — 实时 CPU 使用率、内存占用、磁盘空间、系统负载、进程信息
- **系统设置** — 修改管理员密码、更换头像、设置学校名称、站点信息

### 关键技术特性

| 特性 | 实现方式 | 说明 |
|------|---------|------|
| 一体机屏幕适配 | 纯黑背景 + 高对比度白色字体（最高 `text-8xl`），无黑暗模式变量依赖 | 教室强光环境下清晰可读 |
| 实时倒计时 | 前端 `setInterval` 定时器 + `date-fns` 精确计算秒差 | 无需 WebSocket 推送，部署零依赖 |
| 教室-考试多对多关联 | `exam_classrooms` 联合主键表（`exam_id`, `classroom_id`） | 一场考试分配多间教室，使用 `INSERT IGNORE` 防重复 |
| JWT 双角色认证 | `role: "admin"` 与 `role: "classroom"` 独立签发，Token Payload 类型守卫 | authMiddleware 拒绝 classroom token，classroomAuthMiddleware 仅放行 classroom token |
| 注册码安全体系 | 16 字符集（去除 I/O/0/1）生成 `XXXX-XXXX` 格式，一码一次性使用 | 管理员批量生成，支持按状态筛选，删除前检查是否已被使用 |
| 安装向导 | 浏览器端 4 步流程：欢迎页 → 环境检测 → 数据库配置 → 初始化安装 | 自动创建 8 张数据库表 + bcrypt 加密管理员密码 + 生成随机 JWT 密钥 |
| 数学函数图标 | 所有图标由三角函数/参数方程生成纯 SVG，无第三方图标库依赖 | 无外部请求，离线环境可用 |
| Token 存储隔离 | 管理员：`examhub-token` Cookie + `ADMIN_TOKEN` sessionStorage；教室端：`examhub-classroom-token` + `CLASSROOM_TOKEN` | BroadcastChannel 跨标签页同步，退出时双向清除 |
| 密码安全 | bcryptjs 12 轮加盐哈希存储，修改密码需验证旧密码 | 符合 OWASP 密码存储建议 |
| 软删除策略 | exams 表 `is_active` 字段标记软删除 | 防止误删数据，保留历史记录 |


## 5. 数据库设计

### 表结构总览（8 张表）

| 表名 | 用途 | 核心字段 | 索引策略 |
|------|------|---------|---------|
| `admins` | 管理员账号 | id, username, password(bcrypt), avatar, created_at | username UNIQUE |
| `exams` | 考试信息 | id, subject, exam_date, duration, location, invigilator, notes, is_active | exam_date 索引用于时间排序 |
| `announcements` | 公告 | id, title, content, is_pinned, is_active | (is_active, is_pinned, created_at) 复合索引 |
| `settings` | 系统设置 (KV) | setting_key, setting_value | setting_key UNIQUE |
| `buildings` | 教学楼 | id, name, created_at | name UNIQUE |
| `registration_codes` | 注册码 | id, code, is_used, used_by_classroom_id, created_at | code UNIQUE, 无外键（避免与 classrooms 表循环依赖） |
| `classrooms` | 教室端账号 | id, room_number, building_name, status(PENDING/APPROVED/REJECTED), registration_code_id, password(bcrypt), created_at | registration_code_id UNIQUE, status 索引 |
| `exam_classrooms` | 考试-教室关联 | exam_id, classroom_id | 联合主键 (exam_id, classroom_id) |

### 设计考量

- **避免循环外键**：`registration_codes.used_by_classroom_id` 不设硬外键约束，避免与 `classrooms.registration_code_id` 形成循环依赖，由应用层保证数据一致性
- **教室状态枚举**：`PENDING → APPROVED / REJECTED` 三态流转，支持管理员随时驳回后重新审核
- **联合主键防重**：`exam_classrooms` 使用联合主键，配合 `INSERT IGNORE` 语法天然防重复分配
- **软删除策略**：`exams.is_active` 和 `announcements.is_active` 标记删除状态，不物理删除数据
- **字符集**：全部表使用 `utf8mb4_unicode_ci`，支持中文和 emoji


## 6. 技术栈与项目结构

### 技术选型

| 层级 | 技术 | 版本 | 选型理由 |
|------|------|------|---------|
| 前端框架 | React + TypeScript | 18 / 5.8 | 组件化开发、类型安全 |

| 后端框架 | Fastify | 4 | 高性能异步、插件体系、原生 TypeScript |

## 8. 后续优化方向

当前版本 v1.1.2 已完成核心功能闭环，以下是已规划的后续迭代方向：

### 公告板优化

- **富文本编辑器** — 公告编辑器从纯文本升级为富文本编辑器（基于 TipTap），支持加粗、列表、颜色标注、图片嵌入，让考试通知更醒目
- **定时发布** — 公告支持设置定时发布/到期自动下架，适合提前编辑考试周注意事项并在指定时间自动上线
- **公告分类** — 支持考试通知、教务通知、紧急通知等分类，首页按分类筛选
- **已读回执** — 教室端查看公告后自动上报已读状态，管理员后台可查看各教室端的公告阅读情况

### 安全性加固

- **HTTPS 强制** — 当前已部署 Let's Encrypt 证书，下一步配置 HSTS 头和 HTTP/2，杜绝中间人攻击
- **速率限制** — 对登录/注册接口添加 `@fastify/rate-limit`，单 IP 每分钟最多 5 次尝试，防止暴力破解
- **双重认证（2FA）** — 管理员登录可选开启 TOTP 二次验证，使用 Google Authenticator 或类似工具生成动态码
- **操作日志审计** — 记录所有管理员关键操作（发布/删除考试、审核教室端、修改设置），支持按时间/操作人/操作类型筛选回溯
- **SQL 注入防护** — 当前已使用参数化查询（mysql2 prepared statements），后续增加 SQL 防火墙中间件做二次拦截
- **XSS 防护** — 对公告内容等用户输入做严格的 HTML 标签白名单过滤（DOMPurify），防止恶意脚本注入
- **CORS 严格模式** — 生产环境配置跨域白名单，仅允许本站域名访问 API，拒绝任意来源的 AJAX 请求
- **数据库备份策略** — 配置 MariaDB 定时自动备份（每日凌晨全量 + 每小时增量），备份文件加密存储

### 更多功能规划

- **考试冲突检测** — 管理员分配教室时，自动检测同一教室同一时段是否已被其他考试占用，提示冲突信息
- **考试数据导出** — 支持按学期/按班级导出考试安排表格（Excel/CSV 格式），方便归档和打印
- **考试成绩管理** — 考试结束后，管理员可录入各科目成绩，学生通过学号查询个人成绩（仅限自己可见）
- **教师排考看板** — 为教师提供个人考试安排看板，展示其所有监考场次的日期、教室、科目汇总
- **多学校/多校区支持** — 一套系统支持多所独立学校的数据隔离，每所学校拥有独立的数据库或命名空间
- **移动端 PWA** — 前端添加 Service Worker + Web App Manifest，支持手机桌面快捷方式打开，离线缓存最近考试数据
- **邮件/短信通知** — 考试安排变更时自动发送邮件或短信通知到相关教师和教室端管理员
- **考试倒计时大屏模板** — 提供多种监考模式主题皮肤（高考风、简约风、校园风），学校按需切换
- **二维码快速登录** — 教室端生成二维码，学生扫码即可在手机上快速查看本教室考试信息，无需手动输入网址

### 功能逻辑优化

- **考试状态实时更新** — 当前前端定时轮询刷新，后续改用 Server-Sent Events (SSE) 实现服务端主动推送，每秒精确同步考试状态
- **批量操作增强** — 考试管理列表支持多选批量删除、批量修改时间、批量分配教室
- **教室排期可视化** — 用日历/时间轴视图直观展示每间教室的考试占用情况，管理员拖拽即可调整
- **智能排考建议** — 输入待排考试的科目、年级、班级数量和可用教室列表，系统自动生成最优排考方案（最小化教室冲突和空闲）
- **公告模板库** — 提供常用考试通知模板（考前须知、考场规则、作弊处理办法等），管理员一键套用
- **数据大屏** — 独立的管理数据大屏页面，实时展示全校考试动态（正在进行 X 场、今日共计 X 场、教室使用率图表）
- **前端性能优化** — 组件懒加载（React.lazy）、打包分析（rollup-plugin-visualizer）、图片压缩（vite-plugin-imagemin）
- **无障碍访问（A11y）** — 添加 ARIA 标签、键盘导航支持、屏幕阅读器适配，让视障师生也能流畅使用
- **多语言国际化** — 前端接入 i18n 方案，支持中英双语切换，为双语学校和国际学校提供便利
- **单元测试 + E2E 测试** — 后端添加 Vitest 单元测试覆盖核心路由，前端添加 Playwright 端到端测试覆盖关键用户流程
