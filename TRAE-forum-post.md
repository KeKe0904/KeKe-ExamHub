## 1. 创意名称 + 创意介绍

**创意名称：** KeKe ExamHub — 学校考试信息数字化管理平台

**想解决什么问题：**

当前中国绝大多数中小学的考试信息传递仍停留在"粉笔+黑板"时代。教务处老师在考试周需要逐个教室手写考试安排表，字体不统一、排版靠经验、信息容易写错。一旦考试时间或科目调整，就必须重新擦、重新写，耗时且低效。

更关键的矛盾在于：近年来国家大力推进教育信息化，几乎所有教室都已配备交互式一体机（希沃、鸿合等），但这些设备 90% 时间仅仅充当"高清投影仪"——上课时投屏 PPT，下课后挂一个静态的电子班牌，数字化价值远未释放。考试信息展示这个高频场景，恰恰没有一款专用软件来承接。

**为什么会想到做这个：**

灵感来自一次校园实地观察。一所初中在期末考期间，每个教室门口贴了一张 A4 纸手写通知，上面写着"语文 8:00-10:00，数学 14:00-16:00"，字迹潦草、纸张被风吹得卷边。走到隔壁教室发现同一张纸上涂改了三处时间。而在这些教室的正前方，一块 86 英寸的触控一体机正在循环播放"欢迎各位同学返校"的静态画面。

那一刻的想法很朴素也很直接：为什么不用这块屏幕来显示考试信息？它能动态更新、字体清晰统一、还能倒计时。于是 KeKe ExamHub 诞生了。

**大概是什么产品：**

一个全栈 Web 系统（网站形态），前端适配 PC / 平板 / 手机 / 教室一体机四种尺寸，后端一键部署、图形化初始化，无需任何代码能力即可上线使用。

## 2. 目标用户及痛点

**面向哪些用户：**

| 用户角色 | 核心需求 | 使用设备 |
|---------|---------|---------|
| 教务处 / 管理员 | 统一发布考试、掌握全局安排、管理教室资源 | 电脑（后台管理） |
| 监考老师 / 班主任 | 快速查看自己所负责教室的考试安排 | 手机 / 一体机 |
| 学生 | 随时查询自己的考试时间、地点和注意事项 | 手机 |
| 教室一体机 | 作为固定终端，自动展示本教室被分配的考试信息 | 一体机大屏 |

**在什么场景下使用：**

- **考前准备期（T-7 天）：** 管理员在后台批量录入考试信息，为每场考试分配教学楼和教室，生成注册码供教室端激活。公众首页自动展示考试汇总。
- **考试当天（T-0）：** 每间教室的一体机打开浏览器登录教室端，自动显示本教室当天的考试科目、时间区段、监考老师。页面 30 秒自动刷新，无需人工干预。
- **考试进行中：** 教室端可一键切换为**监考模式** — 全屏显示超大号倒计时、考试科目、剩余时间进度条。当剩余时间低于 5 分钟时，自动触发红色脉冲预警，提示监考老师准备收卷。
- **考试调整期：** 若因特殊情况需要临时调整时间或教室，管理员在后台修改后，所有教室端和公众首页实时同步生效，无需重新通知。
- **日常教务管理：** 管理员可随时管理教学楼信息、生成/撤销注册码、审核新教室端的注册申请。

**当前痛点（如果没有这个产品，用户会面临什么）：**

1. **信息一致性灾难：** 纸质通知在多间教室分散手写，板书字迹因人而异，信息出错率极高。一旦临时调整，需要派人跑遍全楼修改，往返耗时可能超过 20 分钟。
2. **教室一体机资源闲置：** 价值上万元的一体机设备，每天仅在上课时段使用投影功能，其余时间要么黑屏要么播放无意义循环画面，投入产出比极低。
3. **沟通效率低下：** 考试通知依赖教师群、班级群、家长群层层转发，信息在"传话游戏"中逐步失真，学生和家长频繁私信确认。
4. **无法做考试周全局统筹：** 管理员用 Excel 排表，无法直观判断同一教学楼内是否存在教室冲突、同一教师是否在同一时段被安排到两间考场。

## 3. 价值与意义

**效率提升（量化视角）：**

- 原来发布一次全年级考试安排，需要教务处 1-2 名老师耗时约 2 小时（写通知、跑教室、拍照发群）。使用 ExamHub 后，一人后台操作约 5 分钟即可完成。
- 信息修改从"重写+重贴+重通知"缩短为"后台编辑→全局刷新"，耗时从 30 分钟压降到 10 秒。
- 教室端注册采用注册码制度 + 管理员审核流程，确保只有授权的教室终端才能接入系统，从源头杜绝非法获取考试信息的风险。

**社会价值（开源 + 普惠视角）：**

- 项目完全开源免费（GitHub 公开仓库），降低教育机构数字化门槛。对于经费紧张的乡镇中小学、民办学校而言，无需购买商业考试管理系统（年费通常在数千至数万元），即可拥有数字化考试信息管理能力。
- 注册码 + 审核机制保障信息安全，避免考试安排被无关人员获取或篡改。管理员对每一间接入的教室端具有完全控制权（通过/驳回/删除）。
- 网页安装向导实现"零技术门槛部署"：只需在服务器上运行一条脚本，后续在浏览器中填写数据库连接信息即可自动完成表结构创建和管理员账号初始化。一个信息技术老师 10 分钟即可完成全系统上线。

## 4. 核心功能架构

**系统分为三层入口：**

```
                      KeKe ExamHub
                           |
          ┌────────────────┼────────────────┐
     公众前台          教室端大屏          后台管理
      (任何人)      (一体机 + 监考)      (管理员)
          |                |                  |
    考试信息展示     教室考试展示        考试 CRUD
    状态筛选搜索     全屏监考模式        教学楼管理
    公告实时同步     自动刷新          注册码批量生成
    学校信息展示     预警提示(5分钟)     教室端审核流程
                                      公告/系统设置管理
                                      服务器运行监控
```

**关键技术特性：**

| 特性 | 实现方式 |
|------|---------|
| 一体机适配 | 纯黑背景 + 高对比度字号（最高 `text-8xl`），强光下清晰可读 |
| 实时倒计时 | 前端 `setInterval` + `date-fns` 精确计算，无需服务端推送 |
| 教室-考试关联 | `exam_classrooms` 多对多关联表，支持一次考试分配多个教室 |
| JWT 双角色认证 | 管理员 token 与教室端 token 独立签发、互斥验证 |
| 注册码安全 | `XXXX-XXXX` 格式，一次性使用，管理员批量生成并支持未使用撤销 |
| 图形化部署 | Web 安装向导自动创建 8 张数据库表 + 初始化管理员账号 |
| 环境自检 | 一键检测 Node.js / npm / PM2 / Nginx / MariaDB / Git 版本和运行状态 |
| 数学函数图标 | 纯 SVG 生成，无需引入图标库，无外部请求依赖 |

## 5. 技术与部署

**技术栈：** React 18 + TypeScript + Vite 6 + TailwindCSS + Zustand + React Router 7（前端） / Fastify 4 + JWT + bcryptjs + MySQL2（后端） / MariaDB 10.3+（数据库） / Nginx + PM2（部署）

**极简部署：**

```bash
git clone https://github.com/KeKe0904/KeKe-ExamHub.git /opt/examhub
cd /opt/examhub && sudo ./deploy.sh          # 自动安装 Node.js/Nginx/MariaDB/PM2
cd api && pm2 start npm --name examhub-api -- run start
# 浏览器打开 http://你的IP → 安装向导 → 完成！
```
