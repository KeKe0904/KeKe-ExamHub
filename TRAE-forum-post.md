## 1. 创意名称 + 创意介绍

**创意名称：** KeKe ExamHub — 学校考试信息数字化管理平台

**想解决什么问题：**
目前大多数学校的考试信息通过纸质黑板、手写通知或微信群传递，信息更新慢、易出错、查找不方便，尤其是在教室一体机上缺乏统一的考试信息展示方案。

**为什么会想到做这个：**
每次大考前，教务处的老师需要用粉笔在各个教室写考试安排表，字体不统一、信息容易写错，而且一旦考试时间调整就要重写。同时，教室里的交互式一体机大部分时间只当投影用，没有真正发挥数字化价值。我们希望让学校考试信息展示像数字大屏一样专业、高效。

**大概是什么产品：**
一个全栈 Web 系统，前端适配 PC/移动端/教室一体机，后端一键部署，集成数据库支持，面向全校师生使用。

---

## 2. 目标用户及痛点

**面向哪些用户：**
- **学校教务处 / 管理员** — 发布和管理考试安排
- **班主任 / 监考老师** — 快速查看监考安排
- **学生** — 随时查阅考试时间、地点和注意事项
- **教室一体机** — 教室大屏自动显示本教室当天的考试信息

**在什么场景下使用：**
- 考试周前：学生通过手机或网站浏览自己的考试安排
- 考试当天：教室一体机自动展示该教室的考试科目、时间、监考老师、倒计时
- 考试进行中：教室端可一键切换为**监考模式**，全屏显示考试倒计时和考场规则
- 日常管理：管理员在后台上传考试、分配教室、管理教学楼和教室端账号

**当前痛点：**
- 纸质通知容易写错、更新需要重新书写
- 微信群通知容易淹没在聊天记录里
- 教室一体机缺乏专用的考试信息展示软件
- 考试信息管理依赖 Excel 表格，缺乏统一入口

---

## 3. 价值与意义

**效率提升：**
管理员一键发布考试到全平台，教室端自动显示被分配的考试，信息同步实时生效。相比传统手写通知，效率提升显著。

**社会价值：**
为学校提供免费的考试信息化管理方案，降低教育机构数字化转型门槛。教室端注册需管理员审核，保障信息安全。特别适合中小学校、培训机构使用。

---

## 4. 核心功能亮点

| 模块 | 功能 |
|------|------|
| 公众首页 | 考试信息展示、状态筛选、公告栏、倒计时 |
| 教室端大屏 | 一体机专用暗色大屏，自动显示本教室被分配的考试，30 秒自动刷新 |
| 监考模式 | 全屏考试倒计时 + 进度条，最后 5 分钟红色预警，支持全屏 API |
| 考试管理 | 发布/编辑/删除考试，按教学楼分组分配教室，批量分配 |
| 教室审核 | 注册码制度，管理员审核教师端注册申请（通过/驳回） |
| 教学楼管理 | 创建/编辑/删除教学楼，支持删除前关联检查 |
| 注册码生成 | 批量生成注册码（XXXX-XXXX 格式），支持按状态筛选 |
| 图形化安装 | 网页安装向导，一键初始化数据库和配置文件 |
| 服务器监控 | 实时 CPU/内存/磁盘/负载监控 |

## 5. 技术栈

- **前端：** React 18 + TypeScript + Vite 6 + TailwindCSS + Zustand + React Router 7
- **后端：** Fastify 4 + TypeScript + JWT + bcryptjs
- **数据库：** MySQL/MariaDB（8 张表）
- **服务器：** Nginx + PM2 + Node.js
- **设计风格：** 黑白极简，数学函数生成 SVG 图标
